# 📊 SQL Data Cleaning and Exploratory Data Analysis (EDA) using MySQL

This repository showcases an end-to-end SQL project focused on Data Cleaning and Exploratory Data Analysis (EDA) using MySQL.

The project uses a real-world layoffs dataset to demonstrate how raw data can be transformed into a clean, reliable, and analysis-ready dataset. After performing the data cleaning process, exploratory data analysis was conducted to uncover meaningful business insights and trends.

The primary objective of this project was to strengthen practical SQL skills while learning industry-standard data cleaning techniques and analytical approaches commonly used by Data Analysts.

---

# 🎯 Project Objectives

- Clean and prepare raw data for analysis.
- Improve data quality by identifying and fixing inconsistencies.
- Perform exploratory data analysis to identify trends and patterns.
- Generate meaningful business insights using SQL.
- Apply industry-standard SQL practices for data transformation and analysis.

---

# 🛠️ Project Workflow

The project was completed in two major phases.

## Phase 1 – Data Cleaning

The following data cleaning techniques were performed:

- Removed duplicate records
- Standardized inconsistent values
- Handled NULL and blank values
- Converted data into appropriate data types
- Removed unnecessary rows and columns
- Prepared the dataset for analysis

---

## Phase 2 – Exploratory Data Analysis (EDA)

After cleaning the dataset, SQL queries were used to analyze:

- Total layoffs by company
- Layoffs by industry
- Layoffs by country
- Layoffs over time
- Companies with the highest layoffs
- Funding raised vs layoffs
- Year-wise layoff trends
- Additional business insights from the dataset
